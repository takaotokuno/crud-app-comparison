# 要件定義: 商品在庫管理CRUD

## 1. 目的

複数の技術構成で同じ題材のCRUDアプリケーションを実装し、フレームワーク、言語、ORM、API設計、UI実装、認証方式、テスト方針の違いを比較しやすくするための共通仕様を定義する。

題材は「商品在庫管理」とし、商品マスタ、在庫残高、在庫増減履歴を管理する基本的な業務アプリケーションを対象にする。

## 2. 想定ユーザーと権限

| ロール | 想定ユーザー | 主な操作 |
| --- | --- | --- |
| admin | 管理者 | ユーザー管理、商品CRUD、在庫CRUD、在庫取引登録、削除を含む全操作。 |
| staff | 在庫担当者 | 商品閲覧、在庫閲覧、在庫取引登録、在庫調整。 |
| viewer | 閲覧者 | 商品一覧、商品詳細、在庫状況、在庫取引履歴の閲覧のみ。 |

ログイン機能を設け、ログイン済みユーザーだけが商品在庫管理機能を利用できる。初期比較では高度な認可制御は必須にしないが、最低限「未ログインユーザーは管理画面へアクセスできない」ことを必須とする。

## 3. スコープ

### 3.1 対象範囲

- ログイン、ログアウト、ログイン状態の保持
- 商品の作成、一覧表示、詳細表示、更新、削除
- 在庫レコードの作成、一覧表示、詳細表示、更新、削除
- 入庫、出庫、調整などの在庫取引履歴の登録と表示
- 商品、在庫、在庫取引の検索、絞り込み、並び替え
- 入力値検証とエラー表示
- 基本的な永続化
- 基本的な自動テスト

### 3.2 対象外

- パスワードリセット、メール認証、多要素認証
- 複数拠点の在庫管理
- 仕入・販売・出荷などの詳細な業務フロー
- 在庫移動履歴の厳密な会計管理
- 画像アップロード
- バーコード連携
- 外部API連携
- 多言語対応

## 4. 用語

| 用語 | 説明 |
| --- | --- |
| ユーザー | ログインして商品在庫管理機能を利用する人。 |
| 商品 | 在庫管理の対象となる品目。SKU、商品名、価格などの商品マスタ情報を持つ。 |
| SKU | 商品を一意に識別する管理コード。ユーザーが入力する業務上の識別子。 |
| 在庫 | 商品ごとの現在在庫数と安全在庫数を持つ残高情報。 |
| 在庫取引 | 入庫、出庫、調整など、在庫数を増減させる操作の履歴。 |
| 安全在庫数 | 在庫不足を警告するためのしきい値。 |
| 商品ステータス | 商品が販売中、停止中、廃番のいずれかであることを示す状態。 |

## 5. データモデル

データは `User`, `Product`, `Stock`, `StockTransaction` に分けて管理する。特に商品マスタと在庫残高、在庫増減履歴を同一テーブルに混在させない。

### 5.1 User

| フィールド | 型 | 必須 | 制約 | 説明 |
| --- | --- | --- | --- | --- |
| id | UUID または Auto Increment ID | Yes | 一意、変更不可 | システム内部ID。 |
| email | string | Yes | 一意、最大255文字、メール形式 | ログインID。 |
| password_hash | string | Yes | 平文保存禁止 | ハッシュ化済みパスワード。 |
| name | string | Yes | 1〜100文字 | 表示名。 |
| role | enum | Yes | `admin`, `staff`, `viewer` | ユーザーロール。 |
| created_at | datetime | Yes | 作成時に自動設定 | 作成日時。 |
| updated_at | datetime | Yes | 更新時に自動更新 | 更新日時。 |

### 5.2 Product

| フィールド | 型 | 必須 | 制約 | 説明 |
| --- | --- | --- | --- | --- |
| id | UUID または Auto Increment ID | Yes | 一意、変更不可 | システム内部ID。 |
| sku | string | Yes | 一意、1〜32文字、英数字・ハイフン・アンダースコアのみ | 業務上の商品コード。 |
| name | string | Yes | 1〜100文字 | 商品名。 |
| description | string | No | 最大1000文字 | 商品説明。 |
| category | string | No | 最大50文字 | 商品カテゴリ。 |
| price | integer | Yes | 0以上 | 税抜または税込の単価。通貨はJPY、最小単位は円。 |
| status | enum | Yes | `active`, `inactive`, `discontinued` | 商品ステータス。 |
| created_at | datetime | Yes | 作成時に自動設定 | 作成日時。 |
| updated_at | datetime | Yes | 更新時に自動更新 | 更新日時。 |

### 5.3 Stock

| フィールド | 型 | 必須 | 制約 | 説明 |
| --- | --- | --- | --- | --- |
| id | UUID または Auto Increment ID | Yes | 一意、変更不可 | システム内部ID。 |
| product_id | UUID または integer | Yes | Productへの外部キー、一意 | 対象商品。1商品につき在庫レコードは1件とする。 |
| quantity | integer | Yes | 0以上 | 現在在庫数。 |
| safety_stock | integer | Yes | 0以上 | 在庫不足判定のしきい値。 |
| created_at | datetime | Yes | 作成時に自動設定 | 作成日時。 |
| updated_at | datetime | Yes | 更新時に自動更新 | 更新日時。 |

### 5.4 StockTransaction

| フィールド | 型 | 必須 | 制約 | 説明 |
| --- | --- | --- | --- | --- |
| id | UUID または Auto Increment ID | Yes | 一意、変更不可 | システム内部ID。 |
| product_id | UUID または integer | Yes | Productへの外部キー | 対象商品。 |
| stock_id | UUID または integer | Yes | Stockへの外部キー | 対象在庫レコード。 |
| type | enum | Yes | `inbound`, `outbound`, `adjustment` | 在庫取引種別。 |
| quantity_delta | integer | Yes | 0以外 | 在庫増減数。入庫は正、出庫は負、調整は正負どちらも可。 |
| quantity_after | integer | Yes | 0以上 | 取引反映後の在庫数。 |
| reason | string | No | 最大255文字 | 取引理由やメモ。 |
| created_by | UUID または integer | No | Userへの外部キー | 操作ユーザー。認証実装が簡易な場合は未設定可。 |
| created_at | datetime | Yes | 作成時に自動設定 | 取引日時。 |

### 5.5 ステータス・種別定義

| 種別 | 値 | 表示名 | 説明 |
| --- | --- | --- | --- |
| 商品ステータス | active | 販売中 | 通常管理対象の商品。 |
| 商品ステータス | inactive | 停止中 | 一時的に販売または利用を停止している商品。 |
| 商品ステータス | discontinued | 廃番 | 今後取り扱わない商品。 |
| 在庫取引種別 | inbound | 入庫 | 在庫を増やす操作。 |
| 在庫取引種別 | outbound | 出庫 | 在庫を減らす操作。 |
| 在庫取引種別 | adjustment | 調整 | 棚卸しなどで在庫数を補正する操作。 |

## 6. 認証要件

### 6.1 ログイン

- メールアドレスとパスワードでログインできる。
- ログイン成功後は商品一覧画面へ遷移する。
- ログイン失敗時は認証エラーメッセージを表示する。
- パスワードは平文保存せず、各構成で標準的なハッシュ方式を使用する。

### 6.2 ログアウト

- ログイン中のユーザーはログアウトできる。
- ログアウト後はログイン画面へ遷移する。
- ログアウト後に管理画面URLへ直接アクセスした場合はログイン画面へ誘導する。

### 6.3 セッション・トークン

- 画面中心の構成ではセッションCookie方式を基本とする。
- API中心の構成ではCookieセッションまたはBearerトークン方式のどちらでもよい。
- 採用した認証方式は各実装メモに明記する。

## 7. 画面要件

### 7.1 ログイン画面

- メールアドレスとパスワードの入力欄を表示する。
- ログインボタンを表示する。
- 認証失敗時はエラーメッセージを表示する。

### 7.2 商品一覧画面

- 商品と在庫情報を結合し、表形式またはカード形式で一覧表示する。
- 表示項目は SKU、商品名、カテゴリ、価格、現在在庫数、安全在庫数、商品ステータス、在庫更新日時とする。
- 各商品に詳細、編集、削除、在庫取引登録の導線を用意する。
- 新規登録画面への導線を用意する。
- 現在在庫数が安全在庫数以下の商品は視覚的に分かるように表示する。
- データが0件の場合は空状態メッセージを表示する。

### 7.3 商品詳細画面

- 1件の商品情報、在庫情報、直近の在庫取引履歴を表示する。
- 一覧、編集、削除、在庫取引登録への導線を用意する。
- 作成日時と更新日時を表示する。

### 7.4 商品登録画面

- Product と初期 Stock の入力フォームを表示する。
- 登録成功時は Product と Stock を作成する。
- 登録成功後は詳細画面または一覧画面へ遷移する。
- 入力エラーがある場合は、該当項目の近くにエラーメッセージを表示する。

### 7.5 商品編集画面

- 既存の商品情報をフォームに初期表示する。
- 更新成功後は詳細画面または一覧画面へ遷移する。
- SKUを編集可能にする場合でも、一意制約を維持する。

### 7.6 在庫編集・在庫取引登録画面

- 現在在庫数、安全在庫数を編集できる。
- 入庫、出庫、調整のいずれかを選び、在庫取引を登録できる。
- 在庫取引登録時は Stock の `quantity` を更新し、StockTransaction を作成する。
- 出庫後の在庫数が0未満になる操作は登録できない。

### 7.7 商品削除

- 削除前に確認を行う。
- 削除成功後は一覧画面へ遷移する。
- Product を削除する場合、関連する Stock と StockTransaction の扱いを実装メモに明記する。
- 物理削除を基本とする。ただし、構成比較のために論理削除を採用する場合は実装メモに明記する。

## 8. CRUD操作要件

| 対象 | 操作 | 要件 |
| --- | --- | --- |
| Auth | Login | 正しいメールアドレスとパスワードでログインできる。 |
| Auth | Logout | ログイン状態を破棄できる。 |
| Product | Create | 必須項目と制約を満たす商品を登録できる。登録時に初期Stockも作成する。 |
| Product | Read List | 商品一覧を在庫情報付きで取得できる。検索、絞り込み、並び替え、ページングに対応する。 |
| Product | Read Detail | ID指定で商品、在庫、在庫取引履歴を取得できる。存在しない場合は404相当を返す。 |
| Product | Update | ID指定で商品を更新できる。存在しない場合は404相当を返す。 |
| Product | Delete | ID指定で商品を削除できる。存在しない場合は404相当を返す。 |
| Stock | Update | IDまたはproduct_id指定で現在在庫数と安全在庫数を更新できる。 |
| StockTransaction | Create | 入庫、出庫、調整を登録し、Stock.quantityに反映できる。 |
| StockTransaction | Read List | 商品別または全体の在庫取引履歴を取得できる。 |

## 9. 検索・絞り込み・並び替え・ページング

### 9.1 検索

- `q` パラメータで SKU、商品名、説明を部分一致検索できる。
- 大文字小文字の扱いは利用するDBや実装方式に依存してよいが、実装メモに明記する。

### 9.2 絞り込み

- `status` で商品ステータスを絞り込める。
- `category` でカテゴリを絞り込める。
- `low_stock` が true の場合、`Stock.quantity <= Stock.safety_stock` の商品だけを表示する。
- `transaction_type` で在庫取引種別を絞り込める。

### 9.3 並び替え

- 商品一覧の並び替え対象は `sku`, `name`, `price`, `quantity`, `updated_at`, `created_at` とする。
- 在庫取引履歴の並び替え対象は `created_at`, `type`, `quantity_delta` とする。
- 昇順と降順を指定できる。
- デフォルトは商品一覧では `updated_at` の降順、在庫取引履歴では `created_at` の降順とする。

### 9.4 ページング

- ページ番号と1ページあたりの件数を指定できる。
- デフォルトは1ページ目、20件表示とする。
- 1ページあたりの最大件数は100件とする。

## 10. バリデーション

| 対象 | 項目 | ルール | エラーメッセージ例 |
| --- | --- | --- | --- |
| User | email | 必須、一意、メール形式、最大255文字 | メールアドレスを正しく入力してください。 |
| User | password | 必須、8文字以上 | パスワードは8文字以上で入力してください。 |
| User | name | 必須、1〜100文字 | 名前を100文字以内で入力してください。 |
| Product | sku | 必須、一意、1〜32文字、英数字・ハイフン・アンダースコアのみ | SKUは英数字、ハイフン、アンダースコアで32文字以内で入力してください。 |
| Product | name | 必須、1〜100文字 | 商品名を100文字以内で入力してください。 |
| Product | description | 任意、最大1000文字 | 説明は1000文字以内で入力してください。 |
| Product | category | 任意、最大50文字 | カテゴリは50文字以内で入力してください。 |
| Product | price | 必須、0以上の整数 | 価格は0以上の整数で入力してください。 |
| Product | status | 必須、定義済みステータスのみ | ステータスを選択してください。 |
| Stock | quantity | 必須、0以上の整数 | 在庫数は0以上の整数で入力してください。 |
| Stock | safety_stock | 必須、0以上の整数 | 安全在庫数は0以上の整数で入力してください。 |
| StockTransaction | type | 必須、定義済み取引種別のみ | 在庫取引種別を選択してください。 |
| StockTransaction | quantity_delta | 必須、0以外の整数、出庫後在庫が0未満にならないこと | 在庫数の増減値を正しく入力してください。 |
| StockTransaction | reason | 任意、最大255文字 | 理由は255文字以内で入力してください。 |

## 11. API要件

Web APIを実装する構成では、以下のREST APIを標準とする。画面中心の構成でも、内部設計の比較基準として同等の操作を用意する。

| メソッド | パス | 説明 |
| --- | --- | --- |
| POST | `/login` | ログインする。 |
| POST | `/logout` | ログアウトする。 |
| GET | `/me` | ログイン中のユーザー情報を取得する。 |
| GET | `/products` | 商品一覧を在庫情報付きで取得する。 |
| POST | `/products` | 商品と初期在庫を登録する。 |
| GET | `/products/{id}` | 商品詳細、在庫、在庫取引履歴を取得する。 |
| PUT または PATCH | `/products/{id}` | 商品を更新する。 |
| DELETE | `/products/{id}` | 商品を削除する。 |
| GET | `/stocks` | 在庫一覧を取得する。 |
| GET | `/stocks/{id}` | 在庫詳細を取得する。 |
| PUT または PATCH | `/stocks/{id}` | 在庫を更新する。 |
| GET | `/stock-transactions` | 在庫取引履歴を取得する。 |
| POST | `/stock-transactions` | 在庫取引を登録し、在庫数へ反映する。 |

### 11.1 レスポンス方針

- 成功時は対象データまたは処理結果を返す。
- 未ログインの場合は401相当を返す。
- 権限不足の場合は403相当を返す。
- バリデーションエラーは400または422相当を返す。
- 対象データが存在しない場合は404相当を返す。
- SKU、メールアドレスなどの一意制約違反は409または422相当を返す。
- 予期しないエラーは500相当を返す。

### 12.1 ユーザー

## 13. テスト要件

最低限、以下を確認する。

- 正しい認証情報でログインできる。
- 誤った認証情報ではログインできない。
- 未ログイン状態で管理画面または保護APIへアクセスできない。
- 商品と初期在庫を登録できる。
- 必須項目が不足している場合、登録できない。
- SKUが重複している場合、登録または更新できない。
- 商品一覧を在庫情報付きで取得できる。
- 商品詳細で在庫と在庫取引履歴を確認できる。
- 商品を更新できる。
- 商品を削除できる。
- 在庫数と安全在庫数を更新できる。
- 入庫、出庫、調整の在庫取引を登録できる。
- 出庫後の在庫数が0未満になる在庫取引を登録できない。
- 存在しないIDを指定した場合、404相当になる。
- 在庫不足の商品を絞り込める。
